A2 · DI Strategy — module เดิม behavior ต่าง

ไม่ใช่ on/off แต่เป็น polymorphism → port เดียว + impl ต่อ customer

code
# port เดียว — ทุก impl signature เหมือนกัน
abstract class Pricing { abstract calc(o):number }
class DefaultPricing implements Pricing {..}
class DensoPricing   implements Pricing {..ต่าง..}
# bind ตาม brand จุดเดียว (registry ถ้าหลายเจ้า)
providers:[{ provide:Pricing,
  useClass: STRAT[cfg.brand] ?? DefaultPricing }]
# consumer แค่ inject port — ไม่มี if
constructor(private pricing: Pricing) {}
เลือกระดับ (ladder)
● ต่างแค่ ค่า → inject config (ไม่ต้อง strategy)
● ต่าง logic + ลูกค้า ≥2-3 → port + impl (นี่)
● ลูกค้าเดียว → if ก่อน, ดึง strategy ตอนที่ 2 (rule of three)
● ต้องมี interface เหมือนกัน (contract ร่วม)
ยัง 1 image (bind runtime) = ร่ม A · กัน IP ของ impl → B
